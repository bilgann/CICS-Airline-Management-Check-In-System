=============================================================================
DATA FLOW:
=============================================================================

FLTS (state='I') 
  → User enters name, passport, origin, dest, dates, trip type
  
FLTS (state='M' validation complete)
  → XCTL to FLSR with COMMAREA(79) containing name, passport, travel data

FLSR (state='S' then 'O')
  → User selects outbound flight
  
FLSR (state='R' if round-trip)
  → User selects return flight
  
FLSR → XCTL to BOOK with COMMAREA(79) state='B'

BOOK (state='B')
  → Displays formatted confirmation with:
     - Passenger name and passport
     - Formatted dates (MMM DD YYYY)
     - Airport codes with visual route line
     - Formatted times (HH:MM)
     - Bordered flight sections

=============================================================================
COMMAREA STRUCTURE (79 BYTES):
=============================================================================

Byte   Length  Field          Description
-----  ------  -------------  ---------------------------
01     1       CA-STATE       Current state/screen
02-19  18      CA-NAME        Passenger name
20-35  16      CA-PASSPORT    Passport number  
36-38  3       CA-ORIG        Origin airport code
39-41  3       CA-DEST        Destination airport code
42-49  8       CA-DEPDATE     Departure date (YYYYMMDD)
50-57  8       CA-RETDATE     Return date (YYYYMMDD)
58     1       CA-TRIPTYPE    'O'=one-way, 'R'=round-trip
59-64  6       CA-OUT-FLT     Outbound flight number
65-68  4       CA-OUT-DEP     Outbound departure time (HHMM)
69-72  4       CA-OUT-ARR     Outbound arrival time (HHMM)
73-78  6       CA-RET-FLT     Return flight number
79-82  4       CA-RET-DEP     Return departure time (HHMM)
83-86  4       CA-RET-ARR     Return arrival time (HHMM)
